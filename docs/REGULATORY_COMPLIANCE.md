# Regulatory Compliance Mapping

## EU AI Act (2025)

### Article 9: Conformity Assessment

**Requirement:** High-risk AI systems undergo conformity assessment before deployment.

**Framework Support:**

| Requirement | Framework Capability |
|-------------|---------------------|
| Bias risk assessment | Fairness metric computation |
| Technical documentation | Automated report generation |
| Testing protocols | Statistical validation methodology |
| Records retention | Audit trail and result storage |

### Annex IV: Technical Documentation

**Report Sections:**
1. Model description (use case, training data, architecture)
2. Protected attributes tested
3. Fairness metrics with statistical significance
4. Limitations and assumptions
5. Remediation recommendations
6. Methodology appendix

### Article 9.5: Post-Market Monitoring

Framework enables ongoing monitoring through:
- Periodic retesting (user-defined frequency)
- Baseline metric establishment
- Drift detection alerts
- Historical trend analysis

## US ECOA (Equal Credit Opportunity Act)

### Regulation B: Disparate Impact

**Requirement:** Monitor lending for disparate impact on protected classes.

**Protected Classes:** Gender, age, race/ethnicity, marital status, national origin

**Metric:** Demographic parity (approval rate disparity)

**Framework Implementation:**
- Tests all protected classes
- Statistical significance testing
- Confidence interval reporting
- Multi-year result storage

## Important Disclaimers

**Framework provides:**
- Statistical fairness assessment
- Documentation for regulatory submissions
- Risk management evidence

**Framework does NOT provide:**
- Legal compliance certification
- Guarantee of regulatory approval
- Substitute for legal counsel

**Appropriate Use:**
- Internal risk assessment
- Supporting documentation for regulatory filings
- Model governance evidence

**Regulatory Environment:**
Frameworks evolve; users responsible for monitoring regulatory changes and adapting testing approach accordingly.
